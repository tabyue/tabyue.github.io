项目概述

RLinf（Reinforcement Learning Infrastructure） 是清华大学开源的具身/智能体 AI 强化学习基础设施框架。名称中「inf」既代表 Infrastructure（基础设施），也代表 Infinite（无限可能）。

核心使命：为 VLA / VLM / 世界模型的强化学习后训练（Post-Training RL）提供统一、可扩展、分布式的训练 pipeline——是当前具身智能从「SFT 能用」到「RL 好用」的关键工程化基础设施。

一句话定位：RLinf 之于具身 RL，类似 vLLM 之于 LLM 推理。

核心设计理念

  1. 算法-模型-环境解耦：同一份 RL 算法代码可对接任何 VLA 模型和任何仿真/真机环境
  2. 多后端 Actor：根据模型规模自动选择 FSDP / SGLang / vLLM / Megatron 最优推理后端
  3. 混合执行模式：Actor 推理和 Env 交互异步并行 → 吞吐量提升 2.434×
  4. 零代码扩展：用户只需修改 YAML 配置即可扩展到多 GPU / 多节点，无需改 Python 代码

关键算法深度解读

GRPO（Group Relative Policy Optimization）

# RLinf 中 GRPO 的核心实现思路（简化版）
# 1. 对同一初始状态采样 G 条轨迹
# 2. 用 reward 在组内做相对排序
# 3. 优势函数 = (r_i - mean(r_group)) / std(r_group)
# 4. 策略梯度更新

def grpo_loss(log_probs, advantages, old_log_probs, clip_range=0.2):
    ratio = torch.exp(log_probs - old_log_probs)
    clipped = torch.clamp(ratio, 1 - clip_range, 1 + clip_range)
    loss = -torch.min(ratio * advantages, clipped * advantages).mean()
    return loss

为什么 GRPO 适合 VLA 后训练： - PPO 需要 Value 网络估计 baseline，VLA 的状态空间（图像 + 语言）很难训一个好的 Value 网络 - GRPO 用组内相对排序替代 Value 网络，不需要额外网络 - 在 π₀ / GR00T 等大模型上，GRPO 比 PPO 更稳定且容易调参

DSRL（Diffusion Steering via Reinforcement Learning）

原理：不直接 RL 训练巨大的 Diffusion 模型
而是：
1. 冻结预训练好的 Diffusion Policy（如 π₀）
2. 训练一个轻量 SAC Agent（~5M 参数）
3. SAC Agent 在 Diffusion 的 latent noise 空间中「转向」
4. 效果：用极少参数 + 极快训练 实现策略改进

优势：训练成本降低 10-100×（因为不动 Diffusion 主干），适合资源有限的真机 RL

2. 混合执行模式（核心性能优化）

# rlinf/distributed/hybrid_executor.py（概念代码）

class HybridExecutor:
    """
    核心创新：Actor 推理和 Env step 异步并行

    传统串行模式：
      Actor.infer() → Env.step() → Actor.infer() → ...
      总时间 = T_actor + T_env（两者串行）

    RLinf 混合模式：
      while env_batch_1 在执行 step:
          actor 在推理 env_batch_2 的动作
      总时间 ≈ max(T_actor, T_env)（重叠）
      吞吐量提升 = (T_actor + T_env) / max(T_actor, T_env) ≈ 2.434×
    """

    def __init__(self, actor, envs, num_pipeline_stages=2):
        self.actor = actor
        self.envs = envs
        self.stages = num_pipeline_stages

    async def run_episode(self):
        # 双缓冲：两组 env 轮流执行
        obs_buffer = [self.envs[0].reset(), self.envs[1].reset()]

        for step in range(max_steps):
            stage = step % self.stages

            # 当前 stage 的 env 在等 action
            action = await self.actor.infer_async(obs_buffer[stage])

            # 发送 action 给 env（非阻塞）
            env_future = self.envs[stage].step_async(action)

            # 同时处理另一个 stage 的推理
            other_stage = (stage + 1) % self.stages
            obs_buffer[other_stage] = await env_future

3. VLA 模型适配（以 π₀ 为例）

# rlinf/models/vla/pi0_adapter.py（概念代码）

class Pi0Adapter:
    """
    将 π₀ 模型包装为 RLinf 兼容的 Actor

    关键：π₀ 是 Diffusion Policy，输出不是确定性动作
    而是通过去噪 sampling 生成动作序列
    """

    def __init__(self, model_path, backend='sglang'):
        self.model = load_pi0(model_path)
        self.backend = get_backend(backend)

    def get_action(self, obs, instruction):
        # π₀ 的推理：
        # 1. 编码观测 + 语言指令
        # 2. Diffusion 去噪生成动作 chunk
        tokens = self.model.encode(obs, instruction)
        action_chunk = self.model.denoise(
            tokens, 
            num_steps=10,       # diffusion steps
            chunk_size=16       # 预测未来 16 步动作
        )
        return action_chunk

    def get_log_prob(self, obs, instruction, action):
        # RL 需要 log_prob 计算策略梯度
        # 对 Diffusion Policy 来说 = ELBO 近似
        return self.model.compute_elbo(obs, instruction, action)

    def update(self, loss):
        # 反向传播更新模型参数
        loss.backward()
        self.optimizer.step()

4. 真机 RL 数据流

# rlinf/environments/real/franka_env.py（概念代码）

class FrankaRealEnv:
    """
    真机 RL 环境接口

    关键工程问题：
    1. 安全约束（关节限位 / 力矩限制 / 碰撞检测）
    2. 重置策略（人工重置 / 半自动重置）
    3. 奖励设计（学习的 reward 或 基于传感器的 sparse reward）
    4. 延迟补偿（网络延迟 / 控制延迟）
    """

    def __init__(self, robot_ip, camera_type='realsense'):
        self.robot = FrankaInterface(robot_ip)
        self.camera = get_camera(camera_type)
        self.safety = SafetyController()

    def step(self, action):
        # 1. 安全检查
        safe_action = self.safety.clip(action)

        # 2. 发送动作
        self.robot.send_action(safe_action)

        # 3. 等待执行 + 获取观测
        time.sleep(self.dt)  # 控制频率 10-20 Hz
        obs = self._get_obs()

        # 4. 计算奖励
        reward = self.reward_fn(obs)
        done = self.success_detector(obs)

        return obs, reward, done, {}

    def _get_obs(self):
        rgb = self.camera.get_rgb()        # 128×128 RGB
        ee_pos = self.robot.get_ee_pose()  # 7D (pos + quat)
        return {'image': rgb, 'state': ee_pos}

关键论文

1. RLinf 核心论文

  • 标题：RLinf: Reinforcement Learning Infrastructure for Embodied and Agentic AI
  • arXiv：2509.15965
  • 核心贡献：提出混合执行模式 + 算法-模型-环境解耦架构

2. RLinf-USER（真机在线学习）

  • 标题：RLinf-USER: A Unified and Extensible System for Real-World Online Policy Learning in Embodied AI
  • arXiv：2602.07837
  • 核心贡献：首个覆盖「真机数据采集 → 在线 RL 训练 → 策略部署」全链路的统一系统
  • 关键数字：Franka 真机 Peg-insertion 任务，从 0 开始 RL 训练 30 分钟内达到 90% 成功率

3. WoVR（世界模型 RL 抗幻觉）

  • 标题：WoVR: World Models as Reliable Simulators for Post-Training VLA Policies with RL
  • arXiv：2602.13977
  • 核心贡献：
  • 问题：世界模型（Wan / OpenSora）的闭环自回归会产生「幻觉」——想象中成功但现实失败
  • 解法：三层约束体系
    1. 模拟器层：基于 Wan2.2-TI2V-5B + 首帧锚定 + context 噪声注入
    2. 交互层：KIR（关键帧初始化 Rollouts）——从任务关键帧开始想象，缩短误差累积
    3. 对齐层：PACE 共进化——策略和世界模型同步更新，避免分布偏移
