**踩坑教训：**
- **Python print emoji 在 PowerShell GBK 中崩溃**：脚本最后想打印 `⚠️` 触发 `UnicodeEncodeError: 'gbk' codec can't encode character '\u26a0'`。但崩溃之前的 file write 已经全部成功——这是 chr(0x26a0) 加 VS16 (0xfe0f) 复合字符。**今后脚本里要么不用 emoji（用 [WARN] 替代），要么 `import sys; sys.stdout.reconfigure(encoding='utf-8')` 重设编码**。崩溃如果发生在 write 之前会更糟。
- **崩溃后直接重跑 → id 二次插入**：第一次脚本崩在 print 之后所有 write 之后，但我没意识到 write 已成功，又跑了一次。第二次的 title 防重逻辑确实生效了——但因为有 dict ordering 等微妙差异，依然加进了重复条目（实际我现在不能 100% 确认重复来源；可能是上轮就有的）。今后**重跑前必须先 git diff 看是否已有改动入文件**。
- **PowerShell 不识别 `head -30`/`grep`**：要用 PowerShell 原生 `| Select-Object -First 30` 或在 python -c 里直接切片，或者一开始就用 git status --short 不带管道。

- **Python walrus 运算符 `:=` 不能用在 dict 字面量里**：`"content": placeholder := """..."""` 会报 SyntaxError。walrus 仅在表达式上下文合法，dict value 位置不允许。今后写大段长字符串先在外部赋值（如 `ros2_content = """..."""` 然后 `"content": ros2_content`）再放进 dict
